正则表达式

2020-02-25

正则是什么

在编写处理字符串的程序或网页时，经常会有查找符合某些复杂规则的字符串的需要。正则表达式就是用于描述这些规则的工具。换句话说

正则表达式就是记录文本规则的代码。

元字符

常用的元字符有

代码 说明
. 匹配除换行符以外的任意字符
\w 匹配字母或数字或下划线或汉字
\s 匹配任意的空白符
\d 匹配数字
\b 匹配单词的开始或结束
^ 匹配字符串的开始
$ 匹配字符串的结束

字符转义

与其他语言一样，想查找元字符本身的话，需要用转义。

例如，查找\需要用\\

重复

常用的限定符有

代码/语法 说明
* 重复零次或更多次
+ 重复一次或更多次
? 重复一次或零次
{n} 重复n次
{n,} 重复n次或更多次
{n,m} 重复n次或m次

例如

Window\d+可匹配Windows7，windows10

字符类

匹配没有预定义元字符的字符集合
反括号
[0-9]与\d一样

分枝条件

用|把不同的规则分隔开
注意
顺序很重要

\d{5}-\d{4}|\d{5}这个表达式用于匹配美国的邮政编码。美国邮编的规则是5位数字，或者用连字号间隔的9位数字。之所以要给出这个例子是因为它能说明一个问题：使用分枝条件时，要注意各个条件的顺序。如果你把它改成\d{5}|\d{5}-\d{4}的话，那么就只会匹配5位的邮编(以及9位邮编的前5位)。原因是匹配分枝条件时，将会从左到右地测试每个条件，如果满足了某个分枝的话，就不会去再管其它的条件了。

分组

用小括号来指定子表达式，后面跟限定
举例

((2[0-4]\d|25[0-5]|[01]?\d\d?)\.){3}(2[0-4]\d|25[0-5]|[01]?\d\d?)描述了一个IP地址。

反义

常用的反义代码有

代码/语法 说明
\W 匹配不是字母或数字或下划线或汉字的字符
\S 匹配任意的空白符
\D 匹配数字
\B 匹配单词的开始或结束
[^x] 匹配除了x以外的任意字符
[^asdf] 匹配除了asdf以外的任意字符

后向引用

使用小括号指定一个子表达式后，匹配这个子表达式的文本(也就是此分组捕获的内容)可以在表达式或其它程序中作进一步的处理。
认情况下，每个分组会自动拥有一个组号，规则是：从左向右，以分组的左括号为标志，第一个出现的分组的组号为1，第二个为2，以此类推。

  • 分组0对应整个正则表达式
  • 实际上组号分配过程是要从左向右扫描两遍的：第一遍只给未命名组分配，第二遍只给命名组分配－－因此所有命名组的组号都大于未命名的组号

后向引用用于重复搜索前面某个分组匹配的文本。例如，\1代表分组1匹配的文本

可以指定组名

要指定一个子表达式的组名，请使用这样的语法：(?<Word>\w+)(或者把尖括号换成’也行：(?'Word'\w+)),这样就把\w+的组名指定为Word了。要反向引用这个分组捕获的内容，你可以使用\k<Word>

常用分组语法

分类 代码/语法 说明
捕获 (exp) 匹配exp,并捕获文本到自动命名的组里
(?exp) 匹配exp,并捕获文本到名称为name的组里，也可以写成(?'name'exp)
(?:exp) 匹配exp,不捕获匹配的文本，也不给此分组分配组号
零宽断言 (?=exp) 匹配exp前面的位置
(?<=exp) 匹配exp后面的位置
(?!exp) 匹配后面跟的不是exp的位置
(? 匹配前面不是exp的位置
注释 (?#comment) 这种类型的分组不对正则表达式的处理产生任何影响，用于提供注释让人阅读

贪婪与懒惰

  • 当正则表达式中包含能接受重复的限定符时，通常的行为是（在使整个表达式能得到匹配的前提下）匹配尽可能多的字符。这被称为贪婪匹配。
  • 有时，我们更需要懒惰匹配，也就是匹配尽可能少的字符。前面给出的限定符都可以被转化为懒惰匹配模式，只要在它后面加上一个问号?。
    懒惰限定符
代码/语法 说明
*? 重复任意次，但尽可能少
+? 重复一次或更多次，但尽可能少
?? 重复零次或一次，但尽可能少
{n,m}? 重复n到m次，但尽可能少
{n,}? 重复n次以上，但尽可能少

证券

证券交易基础

一，证券交易的概念，基本要素和交易机制

证券交易的概念

已发行的证券在证券市场上买卖的活动
特征有

流动性、收益性和风险性

证券交易的基本要素

证券交易的原则

  1. 公开原则（信息公开）
  2. 公平原则（机会平等）
  3. 公正原则（公正对待）

证券交易的种类

  1. 股票交易
  2. 债券交易
  3. 基金交易
  4. 金融衍生工具交易

证券交易的方式

  1. 现货交易
  2. 远期交易和期货交易
  3. 回购交易
  4. 信用交易

证券交易的有关各方

  1. 证券投资者
  2. 证券公司
  3. 证券交易场所
  4. 证券登记结算机构
  5. 市场中介机构
  6. 市场监管者

证券的交易机制

定期交易和连续交易
指令驱动和报价驱动
证券交易机制目标：

  1. 流动性
  2. 稳定性
  3. 有效性

二，证券交易程序

证券交易程序指投资者通过经纪人在证券交易所买卖证券的交易，要经过开户、委托、竞价成交、结算、过户登记等程序。

证券账户和资金账户

证券账户是证券登记机构为投资者设立
资金账户是证券经纪商为投资者设立的账户

开户的必要性

  1. 了解掌握客户基本情况
  2. 确立委托关系
  3. 防止知情人从中牟利
  4. 方便业务联系

委托

  1. 口头报价
  2. 书面报价
  3. 电脑报价

成交

  • 价格形成方式
    1. 指令驱动方式
    2. 报价驱动方式
  • 竞价方式
    1. 集中竞价
    2. 连续竞价
  • 成立规则

    时间优先，价格优先

  • 竞价结果

结算

买卖双方结清价款和交割证券的过程

证券结算制度

  1. 净额清算制度
  2. 银货对付制度
  3. 共同对手方制度
  4. 分级结算制度

三，委托买卖

委托形式

  • 柜台委托
  • 非柜台委托
  • 委托撤销

委托内容

证券账号、日期、品种、买卖方向、数量、价格、时间、有效期、签名、其他内容

委托价格限制形式

  • 市价委托
  • 限价委托

竞价原则

  • 价格优先
  • 时间优先

Linux系统设置中文版man手册

2019-12-26
Linux

下载中文包

wget https://src.fedoraproject.org/repo/pkgs/man-pages-zh-CN/manpages-zh-1.5.2.tar.bz2/cab232c7bb49b214c2f7ee44f7f35900/manpages-zh-1.5.2.tar.bz2

安装

tar jxvf  manpages-zh-1.5.2.tar.bz2 #解压
cd manpages-zh-1.5.2 #进入解压文件夹
./configure --disable-zhtw #默认安装 
make && make install #编译安装

新建为cman指令来防止抵消man

cd ~
vim .bash_profile

在其中添加

alias cman='man -M /usr/local/share/man/zh_CN'
source .bash_profile

更新bash_profile 使其生效
